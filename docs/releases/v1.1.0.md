# CMGL v1.1.0 Release Notes Draft

CMGL v1.1.0 is a stable release prepared for public package publication. It keeps CMGL local-first and deterministic while making the practical integration path clearer for agent developers.

## Highlights

- Stable top-level API policy for `GovernanceLayer`, `GuardedMemoryBackend`, `AdmissionPolicy`, core models/enums, authority bundle helpers, retrieval filtering, digest helpers, and ledger verification.
- Strict local governance path for persistent memory writes with structured authority bundles.
- Append-only JSONL ledger verification with machine-readable receipts.
- Safe retrieval filtering that blocks stale, superseded, tombstoned, contradicted, quarantined, contaminated, or unauthorized memory.
- Fake-client-tested adapters for Mem0, Graphiti, LangMem, and LangGraph.
- Optional live adapter smoke with skip-on-missing-provider-env default and strict `--require-live-env` mode.
- Release-ready PyPI Trusted Publishing workflow without long-lived PyPI tokens.
- Publishability script covering metadata, required docs, lockfile freshness, CLI smoke, built artifacts, wheel smoke, and manual release gates.

## Installation

After the PyPI publication gate is completed:

```bash
uv add cmgl
```

Before PyPI publication, install from GitHub source:

```bash
uv add "cmgl @ git+https://github.com/kadubon/certified-memory-governance-layer.git"
```

## Security Notes

- Core tests and examples do not require provider keys, LLM calls, cloud services, Neo4j, hosted memory stores, private datasets, or hidden telemetry.
- Live adapter smoke must use GitHub Environment secrets or local environment variables.
- CMGL does not prove factual truth. It proves procedural admissibility under declared policies, records, receipts, and ledger verification.

## Compatibility

- Python: 3.10 through 3.13.
- License: Apache-2.0.
- Package name: `cmgl`.
- Optional extras: `mem0`, `graphiti`, `langmem`, `langgraph`, `adapters`, and `signing`.
